01

先建立心智模型

Claude Code 到底是什么？

它是在终端中运行的“代理式助手”。和只给建议的聊天机器人不同，它能使用工具读取项目、搜索文件、编辑代码、运行测试和调用 Git。

一句话理解

你说清目标，它自己决定先看哪些文件、做哪些修改、运行哪些检查；你可以随时打断、补充或纠正。

它会不断重复一个三步循环

1收集上下文

查看目录、代码、错误信息和项目约定。

→
2采取行动

编辑文件、运行命令，或向你追问关键问题。

→
3验证结果

运行测试、构建或检查差异，不对就继续调整。

它默认能看到什么？

项目文件

当前目录及子目录中的内容。

终端能力

你能运行的构建、测试、Git 等命令。

Git 状态

当前分支、未提交更改和近期历史。

项目说明

CLAUDE.md 中的规则和背景。

从零开始

安装、登录和第一次启动

!
开始前确认三件事

准备一个代码项目、可用的 Claude 订阅或 Console / 企业云访问权限，并确认网络能访问 claude.ai 与 Anthropic API。

PowerShell
irm https://claude.ai/install.ps1 | iex
Terminal
curl -fsSL https://claude.ai/install.sh | bash
PowerShell / CMD
winget install Anthropic.ClaudeCode
  1. 1
    进入项目文件夹

    在终端运行 cd 你的项目路径。Claude Code 会以这个目录为工作范围。

  2. 2
    运行 claude

    首次启动会打开浏览器登录。完成后，凭证会保存在系统中。

  3. 3
    先问，不急着改

    输入：“请用通俗语言说明这个项目做什么、如何启动、主要目录分别负责什么。”

i

03

最重要的能力

把任务说清楚，不必写“魔法提示词”

把 Claude Code 当作一位刚加入项目、能力很强但需要背景信息的同事。好任务通常包含四块：目标、上下文、约束、验收。

目标

你最终想得到什么？

＋
上下文

错误、文件、现状是什么？

＋
约束

哪些不能改？遵循什么规则？

＋
验收

怎样才算完成？跑什么检查？

模糊

“修复登录问题。”

清楚

“登录后刷新页面会退出。请检查 src/auth/ 的会话续期逻辑。先复现并解释原因，再写失败测试、修复并运行相关测试。不要修改数据库结构。”

第一次回答不满意，直接纠正

你不必退出重来。按 Esc 可立即中断；也可以直接补充：“方向不对，问题在会话续期，不是表单校验。先不要改代码，重新分析。”

✦

最实用的提示：让 Claude 有东西可验证。给它测试用例、错误日志、预期输出或 UI 截图，并明确说“完成后运行测试”。

04

照着做即可

六个高频工作流

理解项目

“概览这个代码库，说明入口、核心模块、数据流和启动方式。先不要修改。”

修复错误

“运行测试复现问题，定位根因，先解释再修复；完成后运行相关测试。”

添加功能

“先阅读相关目录并列出计划。等我确认后实现，补测试并检查现有功能。”

重构代码

“保持现有行为不变，先找风险和覆盖缺口，再分小步重构并持续测试。”

编写测试

“找出未覆盖的分支，为正常、边界和错误场景补测试，运行并修复失败项。”

审查改动

“查看当前 git diff，重点检查逻辑错误、安全、兼容性和缺失测试，不要改文件。”

复杂任务使用“两阶段法”

先切换到 Plan 模式，让 Claude 阅读和规划；你确认计划后再让它实现。探索和编码分开，通常更稳。

05

放心使用的关键

权限、检查点与 Git 安全习惯

Manual

编辑文件和运行命令前询问。新手最稳妥。

Accept edits

自动接受文件编辑，其他高影响命令仍会询问。

Plan

只探索和出方案，不修改源文件。

Auto

由后台安全检查评估操作，适合熟悉后使用。

在会话中按 Shift + Tab 切换权限模式。修改前 Claude 会为文件创建 checkpoint；按两次 Esc 可回退文件编辑。

!
Checkpoint 不是万能撤销

它只覆盖文件更改。数据库写入、部署、发消息、调用外部 API 等远程副作用无法靠 checkpoint 恢复，执行前必须看清提示。

提交前做这 4 项检查

  • 要求 Claude 说明改了哪些文件、为什么改。
  • 查看 git diff，不要盲目接受整块变化。
  • 运行相关测试、类型检查或构建。
  • 确认没有密钥、密码、个人数据进入代码或日志。
06

让它懂你的项目

用 CLAUDE.md 保存长期规则

在项目根目录运行 /init，Claude 会引导你创建 CLAUDE.md。它会在以后会话中自动加载，适合存放稳定、简短、可执行的规则。

CLAUDE.md 示例
# 项目概览
- 这是一个 Next.js 电商后台
- 核心代码在 src/，测试在 tests/

# 常用命令
- npm run dev：启动开发环境
- npm test：运行测试
- npm run typecheck：类型检查

# 代码约定
- 使用 TypeScript，避免 any
- 完成改动后运行相关测试
- 未经确认不要修改数据库结构
✓

写什么：每次都要提醒的规则、项目结构、常用命令。
别写什么：密钥、短期任务、长篇教程、很快过时的信息。

07

工作不中断

会话、上下文与恢复

claude -c继续最近会话

在当前目录快速接着上次聊。

claude -r选择旧会话

从历史列表中恢复某次对话。

/clear清空当前上下文

开始一个干净话题。

/compact压缩上下文

保留重点，释放上下文空间。

会话会保存在本地，但不同新会话不会自动继承完整聊天记录。长期规则写进 CLAUDE.md；长任务中可以运行 /context 查看空间被什么占用。

何时开新会话？

话题完全变化、Claude 反复被旧讨论带偏、或上下文已经过于庞杂时。新会话前可让它先总结“已完成、未完成、关键文件和下一步”。

08

学完马上练

60 分钟实战路线

0–10 分钟
认识项目

启动 Claude，询问项目用途、入口、目录结构和运行方式。要求只读、不修改。

10–25 分钟
完成一个小修改

选 README 文案、提示文字或一个简单函数。明确目标和不能改的范围。

25–40 分钟
验证结果

让 Claude 运行相关测试或构建，解释失败原因并修复。

40–50 分钟
复查差异

询问“改了什么、风险是什么”，查看 git diff，必要时要求精简改动。

50–60 分钟
沉淀规则

运行 /init，建立简洁的 CLAUDE.md；为这次任务写清晰提交说明。

🎯
完成标准

你能独立启动 Claude Code、让它先理解项目、下达带验收标准的任务、检查改动并恢复会话，就已经掌握了日常使用的核心。

新手命令与快捷键速查

命令 / 按键作用什么时候用
claude启动交互会话日常进入项目
claude "任务"带着任务启动目标已经明确
claude -p "问题"执行一次查询后退出脚本或快速查询
claude -c继续最近会话接着上次工作
claude -r / /resume选择并恢复会话找回较早任务
/help查看当前可用命令忘记命令时
/init创建项目 CLAUDE.md首次配置项目
/doctor检查安装和配置启动或配置异常
/context查看上下文占用长会话变慢或跑偏
Esc中断当前工作方向不对时立即叫停
Shift + Tab切换权限模式规划、手动确认或自动执行
Ctrl + D / /exit退出 Claude Code结束会话

遇到问题，先这样排查

命令找不到

重开终端，运行 claude --version；仍失败就执行 /doctor 或重新安装。

回答开始跑偏

按 Esc 叫停，说明哪里不对；必要时 /clear 或开新会话。

上下文太满

运行 /context 查看占用，使用 /compact 聚焦当前任务。

改动太大

要求停止、解释 diff、缩小范围；需要时双击 Esc 回退文件。
